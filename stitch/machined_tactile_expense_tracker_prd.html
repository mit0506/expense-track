# Machined Tactile Expense Tracker ## Product Overview **The Pitch:** A high-fidelity, precision-engineered expense tracker that feels like premium audio hardware. Discards soft gradients and illustrations in favor of rigid grids, strict gray palettes, and tactile, light-responsive components. **For:** Power users, financial analysts, and design-conscious individuals who value density, precision, and tool-like utility. **Device:** desktop **Design Direction:** Machined Tactile (Modern Skeuomorphism). Brutally precise, physical-feeling interface driven by lighting angles, 1px top-edge highlights, debossed inputs, and strict typographic hierarchy. **Inspired by:** Teenage Engineering interfaces, Ableton Live, Linear. --- ## Screens - **Dashboard:** High-level metrics, recent activity, and rapid-entry focus. - **Ledger:** Dense, filterable, sortable data table of all transactions. - **Entry Console:** Precision form for logging new expenses with split-pane categorization. - **Telemetry (Analytics):** Bar charts and distribution metrics with physical-feeling toggles. --- ## Key Flows **Logging a Transaction:** Rapid, keyboard-first data entry. 1. User is on Dashboard -> sees global `+ Add Expense` machined button 2. User clicks button (or hits `Cmd+K`) -> Entry Console slides up with physical easing 3. User tabs through debossed inputs -> clicks `Commit` -> button depresses 2px, ledger updates. ---
Design System ## Color Palette - **Primary:** `#F2542D` (Anodized Orange) - Primary action buttons, active states - **Background:** `#111113` (Gunmetal Base) - Main app background - **Surface:** `#1A1A1D` (Machined Plate) - Cards, elevated containers - **Surface Inset:** `#0D0D0F` (Debossed Area) - Input backgrounds, empty states - **Text:** `#EBEBEB` (Titanium White) - Primary data, headings - **Muted:** `#84848B` (Milled Gray) - Labels, inactive icons, borders - **Accent:** `#00E5FF` (LED Cyan) - Hover highlights, data visualization ## Typography - **Headings:** `Archivo`, 600, 24px, `-0.02em` tracking - **Body:** `Archivo`, 400, 14px, `1.4` line-height - **Data/Numbers:** `Space Mono`, 400, 13px, tabular nums - **Labels:** `Space Mono`, 700, 11px, uppercase, `0.05em` tracking **Style notes:** - **Lighting Angle:** Strictly 90-degree top-down. - **Highlights:** `inset 0 1px 0 rgba(255,255,255,0.08)` on all raised surfaces. - **Shadows:** Inputs use `inset 0 2px 4px rgba(0,0,0,0.4)`. Raised buttons use `0 4px 6px rgba(0,0,0,0.5), 0 1px 3px rgba(0,0,0,0.3)`. - **Border Radius:** `4px` global standard. `2px` for inputs. No pill shapes. ## Design Tokens ```css :root { --color-primary: #F2542D; --color-background: #111113; --color-surface: #1A1A1D; --color-surface-inset: #0D0D0F; --color-text: #EBEBEB; --color-muted: #84848B; --color-accent: #00E5FF; --font-sans: 'Archivo', sans-serif; --font-mono: 'Space Mono', monospace; --radius-sm: 2px; --radius-md: 4px; --shadow-raised: 0 4px 6px rgba(0,0,0,0.5), inset 0 1px 0 rgba(255,255,255,0.08); --shadow-pressed: inset 0 2px 4px rgba(0,0,0,0.6); --shadow-input: inset 0 2px 4px rgba(0,0,0,0.4), inset 0 0 0 1px rgba(255,255,255,0.05); } ```

Screen Specifications ### Dashboard **Purpose:** Command center for financial status and quick entry. **Layout:** 240px left sidebar, 1fr main content area, 320px right context panel. **Key Elements:** - **Metric Dials:** 3 horizontal cards (Balance, In, Out). Machined plate style, Space Mono numbers, cyan LED indicators. - **Recent Activity:** 5-row mini ledger. Striped rows (`#1A1A1D` and `#161619`), 1px muted bottom borders. - **Global Add Button:** 48px height, Anodized Orange, sharp 4px radius, raised shadow. **States:** - **Empty:** "NO DATA" stamped in center, Space Mono, `#84848B`, 14px. - **Loading:** Shimmer effect using `rgba(255,255,255,0.02)` sweep over `#1A1A1D`. - **Error:** Red LED (`#FF3366`) border on failed modules, "SYS_ERR" text. **Components:** - **Metric Card:** 240x120px, `#1A1A1D` surface, 1px top highlight, inner value in `Space Mono` 32px `#EBEBEB`. **Interactions:** - **Hover Metric:** 1px `#00E5FF` border swap, 0ms transition. - **Click Add:** Hardware click sound (optional), button depresses via `--shadow-pressed`, text shifts down 1px. ### Ledger **Purpose:** Exhaustive, filterable transaction database. **Layout:** Full-width grid under a rigid 64px control bar. **Key Elements:** - **Control Bar:** Dropdowns for Date, Category, Amount. Raised tactical buttons. - **Data Table:** Dense, 32px row height. Tabular numbers perfectly aligned right. - **Search:** Debossed input `240px` wide, `Space Mono` placeholder "SEARCH_LEDGER...". **States:** - **Empty:** 0 results. "QUERY RETURNED 0 ROWS". - **Loading:** Ghost rows with inset shadows. **Components:** - **Table Header:** 24px height, `#0D0D0F` background, `#84848B` text, 11px uppercase mono. - **Status Tag:** 6px colored dot + text. Orange (Pending), Cyan (Cleared). **Interactions:** - **Hover Row:** Background shifts to `#222226`. - **Click Row:** Opens detail pane sliding from right side (320px width). ### Entry Console **Purpose:** Precision logging tool. **Layout:** Centered modal overlay, 480px width, heavy drop shadow (`0 24px 48px rgba(0,0,0,0.8)`). **Key Elements:** - **Amount Input:** Massive 64px high debossed field, 48px `Space Mono` cyan text. - **Numpad:** 3x4 grid of physical-feeling buttons. `#1A1A1D` surface, 1px gap. - **Category Dial:** Horizontal scrollable list of squared tags. **States:** - **Empty:** Amount at "0.00". - **Error:** Shake animation (2px left/right), input border turns `#FF3366`. **Components:** - **Keypad Button:** 80x64px, `--shadow-raised`, centers text. **Interactions:** - **Click Keypad:** Instantly snaps to `--shadow-pressed`, 100ms return easing. - **Commit:** Slide up modal dismissal with `cubic-bezier(0.16, 1, 0.3, 1)` easing. ### Telemetry (Analytics) **Purpose:** Visual distribution of spending. **Layout:** Split 50/50 horizontal. Top: Master Bar Chart. Bottom: Category breakdowns. **Key Elements:** - **Bar Chart:** Sharp rectangular columns. No rounded tops. Hover reveals exact mono values. - **Time Toggle:** 3-way segmented control (30D, 90D, YTD). Physical sliding tab. **States:** - **Empty:** Axis lines visible, no bars. **Components:** - **Bar:** `#84848B` default, `#F2542D` on hover. 100% sharp edges. **Interactions:** - **Hover Bar:** Bar brightens, tooltip snaps (0ms delay) above cursor with tactile shadow.

Build Guide **Stack:** HTML + Tailwind CSS v3 **Build Order:** 1. **Dashboard** - Establishes the lighting system (inset shadows, highlights) and grid density. 2. **Entry Console** - Defines the interactive "hardware" feel, active states, and inputs. 3. **Ledger** - Sets the standard for tabular data and typography constraints. 4. **Telemetry** - Integrates charting components with the established visual language. **Tailwind Extensions Needed:** - Custom box-shadows for `shadow-raised`, `shadow-pressed`, `shadow-debossed`. - Font-family definitions for `mono` (Space Mono) and `sans` (Archivo). - Custom background colors mapped strictly to the palette (`bg-surface`, `bg-inset`).
